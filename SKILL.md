# Thesis Defense Expert

学术答辩辅助技能，深度分析项目源码和论文内容，生成专业的答辩稿和 PPT 大纲。

## 通用性说明

本技能适用于**任意技术栈**的项目：
- Java Spring Boot / Spring Cloud
- Python Flask / Django / FastAPI
- Node.js Express / NestJS
- Go / Rust / C++ 等

技能会自动识别项目类型并调整分析策略。

---

## 输入

1. **项目源码目录**：默认为当前目录（可指定）
2. **论文文件**：`thesis.md`、论文.pptx、论文.docx 或用户指定路径

---

## 执行流程

### 第一步：确认信息

使用 AskUserQuestion 询问用户：

**问题 1：答辩类型**
- 本科答辩（5-10分钟）
- 硕士答辩（15-20分钟）
- 博士答辩（30-45分钟）

**问题 2：输出格式**
- 答辩稿 + PPT 大纲（推荐）
- 仅 PPT 大纲
- 仅答辩稿

**问题 3：分析范围**
- 分析论文 + 项目源码（推荐）
- 仅论文文件
- 仅项目源码

> **注意**：根据答辩类型自动调整内容深度和时间分配策略

---

### 第二步：提取架构（通用）

**扫描项目结构**
- 使用 Glob 查找源码文件（按语言：Java `**/*.java`, Python `**/*.py`, JS `**/*.js`, TS `**/*.ts`）
- 查找构建配置（pom.xml, build.gradle, package.json, requirements.txt, go.mod 等）
- 查找配置文件（application.properties, application.yml, config.py, .env 等）

**识别核心模块**
- 按框架惯例定位：Controller/Handler/View 层（入口）
- Service/Business Logic 层（业务逻辑）
- Repository/DAO/Model 层（数据访问）
- Entity/DTO/Schema 层（数据模型）

**分析数据模型和数据库设计**
- 查看数据模型类/表结构
- 识别关系（1:1, 1:N, N:M）
- 分析持久化方案（SQL vs NoSQL）

**定位核心业务逻辑**
- 搜索关键词定位：API endpoint, Route, Controller mapping
- 如涉及 AI/LLM：搜索 Vector, Embedding, RAG, LLM, Model, AI, Search, Similarity
- 分析算法和业务流程

**输出第二步结果**
- 项目模块结构（文字描述）
- 数据模型关系图（文字描述）
- 核心业务流程图（文字描述）

---

### 第三步：生成大纲

基于论文结构和源码实现，生成学术性强的答辩材料。

**标准大纲结构**

```
1. 选题背景与意义
   - 研究背景
   - 研究问题
   - 选题动机

2. 关键技术概述
   - 技术选型及理由
   - 核心技术原理

3. 系统设计与实现
   - 整体架构
   - 核心模块详解
   - 数据库设计

4. 亮点与创新
   - 技术亮点
   - 创新点
   - 个人工作量体现

5. 总结与展望
   - 工作总结
   - 未来方向
```

**动态调整**
- 根据论文章节动态调整大纲顺序
- 根据项目实际模块调整技术章节内容
- 根据答辩类型调整各部分内容深度

---

## 附录：答辩技巧提醒

**开场白要点**：
- 简洁自我介绍（姓名、专业、论文题目）
- 直入主题，不说"我的论文写得不好"等套话

**过渡语**：
- "接下来..."
- "需要特别说明的是..."
- "针对这个问题..."

**应对质疑**：
- 保持冷静，感谢质疑
- 诚实承认，合理辩护

**结束语**：
- "以上就是我的汇报，感谢各位老师的聆听！"

## 注意事项

1. **论文文件识别顺序**：
   - 用户指定路径 → `thesis.md` → `thesis.txt` → 论文.pptx → 论文.docx
   - 如遇到 .pptx 或 .docx 文件，使用 pptx skill 或 docx skill 读取内容

2. **项目源码目录**：
   - 默认为当前目录
   - 如有多个项目，提示用户确认目录

3. **语言风格**：
   - 专业、客观、学术化
   - 突出个人工作量和技术难度
   - 避免空洞的套话

4. **代码描述要求**：
   - 用文字清晰描述关键代码逻辑
   - 标注关键类名和方法名
   - 解释数据流和处理流程

5. **PPT 设计建议**：
   - 每页聚焦一个主题
   - 文字简洁，以要点为主
   - 包含可能被问到的技术细节准备

## 附录一：高频Q&A问题库

### 技术类问题

**Q1：你为什么选择XX技术/框架？相比其他方案有什么优势？**
> 回答策略：先肯定其他方案的可行性，再重点说明本项目选择该技术的具体原因（性能、社区支持、学习成本、项目契合度等），避免贬低其他技术。
> - "我对比了A和B方案，考虑到...最终选择了..."
> - "该框架的...特性非常适合我们的...场景"

**Q2：项目的技术难点在哪里？你是如何解决的？**
> 回答策略：诚实承认难点，重点描述解决思路和过程，体现问题分析能力。
> - "在这个项目中，我遇到的主要难点是..."
> - "针对这个问题，我采用了...方案，通过...步骤解决了"

**Q3：系统的性能如何？有做过优化吗？**
> 回答策略：提供具体数据或测试结果，说明优化措施和效果。
> - "经过...优化后，响应时间从...降低到..."
> - "我通过...方式提升了系统性能"

**Q4：如果重新做这个项目，你会如何改进？**
> 回答策略：展示反思能力和成长空间，但要肯定当前工作的合理性。
> - "如果重新来做，我可能会在...方面进行优化"
> - "目前...的设计还有改进空间，我计划在后续工作中..."

### 工作量类问题

**Q5：哪些模块是你独立完成的？如何证明这是你独立完成的？**
> 回答策略：自信、具体地描述个人贡献，可引用具体的代码文件、commit记录、设计文档。
> - "我独立完成了...模块的开发，包括..."
> - "从Git日志可以看出，我贡献了...行代码"

**Q6：你在项目中遇到的最大挑战是什么？**
> 回答策略：选择有代表性的挑战，描述克服过程，体现学习能力。
> - "最大的挑战是...，我通过...方式克服了"

### 学术类问题（硕士/博士）

**Q7：你的创新点是什么？相比现有研究有什么突破？**
> 回答策略：清晰、谦虚地陈述创新点，用具体数据或效果支撑。
> - "本文的创新点主要包括...，相比之前的研究...取得了...的提升"

**Q8：你的研究方法是什么？为什么采用这种方法？**
> 回答策略：说明方法的科学性和适用性。
> - "我采用了...方法，因为...，该方法在...领域被广泛使用"

**Q9：你的工作在未来有什么应用前景？**
> 回答策略：展示工作的实际价值和未来方向。
> - "该工作可以应用于...场景，具有...的实用价值"

## 附录二：答辩技巧指导

### 开场白模板

**推荐开场白**：
> "各位老师好，我是...，我的论文题目是《...》。下面我将从选题背景、系统实现、创新点等方面进行汇报。"

**不推荐**：
- 过于冗长的自我介绍
- "我的论文写得不好，请老师多指正"（过于谦虚）
- "接下来我将从几个方面进行汇报"（太空洞）

### 过渡语整理

- 引入下一个部分："接下来，我来介绍.../下面进入...部分"
- 强调重点："需要特别说明的是.../其中最关键的是..."
- 回应评委："感谢老师的提问，针对这个问题..."

### 应对策略

**当评委质疑你的工作**：
1. 保持冷静，感谢质疑
2. 承认合理批评，说明已知局限
3. 解释原因和当时的选择
4. 如有后续改进计划可补充

**当遇到完全不懂的问题**：
1. 诚实承认："这个问题我了解得不够深入..."
2. 尝试从已知知识推导
3. 表示会在后续学习中完善

**当忘词时**：
1. 不要慌张停顿，直接跳到下一个熟悉的要点
2. 快速掠过当前点，用"总之..."过渡
3. 不要反复说"呃..."

### 结束语模板

**推荐结束语**：
> "以上就是我的汇报，感谢各位老师的聆听和指导！请各位老师批评指正。"

**禁忌**：
- "我的汇报结束了"（过于生硬）
- "谢谢大家，请老师提问"（缺少谦逊）
